# 剧本自动生成器 Web 应用开发需求文档 (PRD)

## 1. 项目概述

本项目旨在开发一个轻量级的内部/朋友间使用的 Web 应用。用户输入四个设定的核心变量，系统通过调用大语言模型（LLM）API，一键生成一篇约 1500 字、具有极强《肖申克的救赎》电影旁白质感的高级短视频剧本。

## 2. 技术栈要求 (极简、轻量、适配现有环境)

* **前端:** 纯原生 HTML + CSS + 原生 JavaScript (无需复杂框架，保证单文件或极简结构即可)。
* **后端:** Python (推荐使用 FastAPI 或 Flask)。
* **部署:** 必须提供 `Dockerfile` 和 `requirements.txt`，以便在已有的 Docker 和 VS Code 环境中一键无脑跑通。

## 3. 核心功能与页面交互 (UI/UX)

页面整体风格要求：极简、暗黑模式（电影高级感）、无多余视觉干扰。

**核心元素：**

1. **标题:** “反重力剧本生成器” (居中，低调的字体)。
2. **输入区 (4个 Input 框，需带有清晰的 Placeholder 提示):**
   * 【时代背景】 (例：古埃及、赛博朋克2077)
   * 【具体地点】 (例：底比斯烂泥潭、霓虹灯下的贫民窟)
   * 【底层身份】 (例：踩泥巴的奴隶、捡垃圾的黑客)
   * 【逆袭元素】 (例：天生吸猫体质、能听懂机械语言)
3. **触发动作:** 一个醒目的按钮，文案为“一键生成宿命”。点击后按钮需有 Loading 状态（防连点）。
4. **输出区:** 一个宽大的只读文本框 (Textarea) 或区块，用于展示生成的 1500 字剧本，支持一键复制。

## 4. 后端核心逻辑与 API 设定

1. **接口设定:** 提供一个 POST 接口 (如 `/api/generate_script`)，接收前端传来的 4 个变量。
2. **LLM 整合:**
   * 在后端代码中预留 LLM API 的调用逻辑（如 OpenAI API 或其他兼容格式的 API，将 API Key 设置为环境变量读取）。
   * **【极其重要】:** 必须在后端代码中硬编码 (Hardcode) 一个 `SYSTEM_PROMPT` 变量。后端在组装请求时，必须将前端传来的 4 个变量拼接到 User Message 中，并受该 `SYSTEM_PROMPT` 严格约束。

**(开发者注：请在后端代码中预留一个名为 `SYSTEM_PROMPT` 的常量，我稍后会将完整的 Prompt 文本填入其中。)**

## 5. 交付物要求

请一次性输出以下文件结构的代码：

1. `app.py` (后端主程序，包含完整的接口路由和带注释的 LLM 调用逻辑)
2. `index.html` (前端页面，内嵌 CSS 和 JS，或拆分为极简的 `style.css` 和 `script.js`)
3. `requirements.txt` (Python 依赖包列表)
4. `Dockerfile` (用于一键构建运行环境)
5. 一段简短的 README 运行指南。
